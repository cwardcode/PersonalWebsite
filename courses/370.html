CS370 - Operating Systems

Project 1

ArrayList in C

This application was used as a way to familiarize ourselves with C. It consists of a simple array-based stack including the standard push, pop, peek, and delete functions.

Project 2

A Simple Shell

A simple shell application for Unix, along the lines of Bash. The shell included the following functionality: ls and rm commands. stdin, stdout, and stderr redirection, along with redirecting both stderr and stdout. Inter-process communication via pipe Signal handler to catch "ctrl-c", so as not to kill the shell itself.

Project 3

Zombie Shotgun

A user-space library with a call to a kernel-based system call for a Minix operating system. Given a PID, the system-call will search the process table, and if the PID is found, will search for any children that are zombies. If a zombie is found, it is eliminated from the process table, effectively "killing" it.

Project 4

Thread-Safe Bounded Circular Buffer

A C-based application that implements a thread-safe bounded circular buffer, as a brief introduction to threading. As a thread attempts to write to the buffer, it must first check the buffer's capacity, and ensure other processes are not writing as well. To read, the buffer must have data, else it will block until data is available.

Extra - Gentoo 1

System Calls in Gentoo

This project consisted of modifying the Gentoo kernel, and added a custom implementation of getpid(), along with added system calls getpppid(), and zombiecount(). getpppid() retrieved the grandparent PID of the input PID, and zombiecount output the number of zombies found for a given PID.

Extra - Gentoo 2

Inter-process Message Queue

This project consisted of modifying the Gentoo kernel with an inter-process message queue for communication within a monolithic kernel. The message queue system is entirely based in kernel space. This introduced the need for not only thread blocking but also the need for using kernel-specific functions.
